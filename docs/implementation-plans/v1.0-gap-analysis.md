**Last Updated:** 2026-03-01

## P0 — Critical for v1.0

### Locator: `get_by_*` Methods

These are *the* modern Playwright locator pattern. Without them, users cannot write idiomatic Playwright tests.

- [x] `get_by_role(role)` — Locate by ARIA role (most important)
- [x] `get_by_text(text)` — Locate by text content
- [x] `get_by_label(text)` — Locate by associated label
- [x] `get_by_placeholder(text)` — Locate by placeholder text
- [x] `get_by_test_id(test_id)` — Locate by test ID attribute
- [x] `get_by_alt_text(text)` — Locate by alt text
- [x] `get_by_title(text)` — Locate by title attribute

These must also be available on **Page** and **Frame** (they delegate to the main frame's locator).

### Locator: Filtering & Composition

- [ ] `filter()` — Narrow locator with additional conditions (has_text, has, etc.)
- [ ] `and_(locator)` — Match both this AND another locator
- [ ] `or_(locator)` — Match either this OR another locator
- [x] `all()` — Return array of locators for each match

### Locator: Missing Interactions

- [ ] `blur()` — Remove focus
- [ ] `focus()` — Set focus
- [ ] `bounding_box()` — Get element dimensions/position
- [ ] `drag_to(target)` — Drag and drop
- [ ] `dispatch_event(type)` — Fire DOM events
- [ ] `evaluate(expression)` / `evaluate_all(expression)` — JS on elements
- [ ] `press_sequentially(text)` — Character-by-character typing
- [ ] `scroll_into_view_if_needed()` — Scroll into viewport
- [ ] `wait_for()` — Wait for element state
- [ ] `tap()` — Touch gesture
- [ ] `is_hidden()` / `is_disabled()` — Missing negative state checks
- [ ] `all_inner_texts()` / `all_text_contents()` — Bulk text retrieval
- [ ] `page` property — Back-reference to owning Page

### Page: Core Missing Methods

- [x] `go_back()` — History back navigation
- [x] `go_forward()` — History forward navigation
- [ ] `set_content(html)` — Set page HTML directly
- [ ] `wait_for_load_state(state)` — Wait for `load/domcontentloaded/networkidle`
- [ ] `wait_for_url(url)` — Wait for navigation to matching URL
- [ ] `set_default_timeout(timeout)` — Default timeout for all operations
- [ ] `set_default_navigation_timeout(timeout)` — Navigation-specific timeout
- [ ] `is_closed` property — Whether page has been closed
- [ ] `frames` property — List all frames

### Page: `get_by_*` Convenience Methods

- [x] `get_by_role(role)`
- [x] `get_by_text(text)`
- [x] `get_by_label(text)`
- [x] `get_by_placeholder(text)`

### BrowserContext: Cookie & State Management

## Recommended Implementation Order

Based on real-world usage patterns and what blocks the most common test scenarios:

### Phase 1: Modern Locator Patterns
`get_by_role`, `get_by_text`, `get_by_label`, `get_by_placeholder`, `get_by_test_id`, `get_by_alt_text`, `get_by_title` on Locator, Page, and Frame. Plus `filter()`, `and_()`, `or_()`.

**Why first:** These are what Playwright recommends for ALL new tests. Without them, the Rust bindings feel like an older version of Playwright.

### Phase 2: Navigation & Synchronization
`go_back`, `go_forward`, `wait_for_load_state`, `wait_for_url`, `set_content`, `set_default_timeout`.

**Why second:** Required for reliable, non-flaky tests.

### Phase 3: Response/Request Body Access
Response `body()`, `text()`, `json()` and Request `headers`, `post_data`, `response()`.

**Why third:** Critical for API testing and network assertions.

### Phase 4: BrowserContext Completeness
Cookie management, context-level routing, permissions, geolocation, offline mode.

**Why fourth:** Required for auth testing and environment simulation.

### Phase 5: FrameLocator & Frame Improvements
FrameLocator class, Frame `get_by_*` methods, iframe support.

**Why fifth:** Many real-world apps use iframes (payment forms, embeds, ads).

### Phase 6: Event System & Waiting
`expect_*` methods, expanded `on()` events, FileChooser, ConsoleMessage.

**Why sixth:** Enables advanced test patterns and debugging.
